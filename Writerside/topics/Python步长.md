# Python步长

最近开始学习`Python`，遇到了一个很有趣的东西，分享一下。

```python
str = '0123456789'
print(str[1:len(str):4])
print(str[0:len(str):3])
print(str[0:len(str):2])
print(str[1:len(str):3])
print(str[1:len(str):2])

159
0369
02468
147
13579
```
这就是`Python`的步长，其实，就是针对字符串进行一个循环的截取操作。只不过`Java`的话会很麻烦，需要循环、获取、判断下标、截取、拼接最后再打印。

`Python`这个就很简单，一行代码搞定，只不过看起来很难理解，实际上一点儿都不难。

以`print(str[1:len(str):3])`这个举例，其他的举栗子好像会被数字给误导。

这个为什么结果会是`147`呢？

首先，我们要先明白一个东西，就是`str[]`这个方法做了什么。`str[]`的方法，里面实际上有三个参数。

```python
str[start, end, step]
第一个参数start很好理解，开始的下标
第二个参数end结束的下标
第三个参数step步长，就是隔几个字符

str[1:len(str):3]
开始是1，也就是从str的数字1开始，结束是字符串的长度。
间隔3个字符串，不包含开始的字符串，但是包含结束的字符串。那，从1开始，不包含1开始，就是从2开始查数，查3个，234，包含结束的字符串，结束的字符串是4，所以，4进来了。
继续，从4后面的5开始，查3个，567。7进来了。
从7后面的8开始查3个，但是不够了。所以，最终结果就是147,1是必须存在的，意思就是start下标开始的字符串，肯定包含它。
```
