什么是常量折叠、死代码消除、循环展开、内联函数

约 800 字大约 3 分钟

JVM面试题

2026-01-05

常量折叠、死代码消除、循环展开、内联函数，这些都是编译优化技术，用于提升程序的执行效率。下面逐个解释：

1. 常量折叠（Constant Folding）

定义：

编译器在编译时直接计算常量表达式的值，而不是在运行时计算。

示例：

// 优化前：运行时计算 2 + 3
int x = 2 + 3;
int y = x * 10;

// 优化后：编译时直接计算结果
int x = 5;  // 2+3=5
int y = 50; // 5*10=50

好处：

  • 减少运行时的计算开销
  • 简化代码结构

2. 死代码消除（Dead Code Elimination）

定义：

移除程序中永远不会执行或执行后对程序结果无影响的代码。

类型1：永远不会执行的代码

// 优化前：if条件恒为false，内部代码永远不会执行
if (1 > 2) {
    System.out.println("这段代码永远不会执行");
}

// 优化后：直接删除死代码
// [无代码]

类型2：执行后无影响的代码

// 优化前：变量a赋值后未被使用
int a = 10;  // 死代码，因为a后续未被访问
int b = 20;
System.out.println(b);

// 优化后：删除无用的变量赋值
int b = 20;
System.out.println(b);

好处：

  • 减小程序体积
  • 提高执行效率（减少不必要的指令）

3. 循环展开（Loop Unrolling）

定义：

将循环体多次重复，减少循环控制（如条件判断、计数器递增）的开销。

示例：

// 优化前：常规循环，每次迭代都需要判断i<10和i++
for (int i = 0; i < 10; i++) {
    sum += i;
}

// 优化后：展开循环，减少5次循环控制开销
for (int i = 0; i < 10; i += 2) {
    sum += i;
    sum += i + 1;  // 手动展开一次迭代
}

好处：

  • 减少循环控制的指令开销
  • 提高指令级并行性（CPU可同时处理更多独立操作）

注意事项：

  • 过度展开会导致代码膨胀（程序体积变大）
  • 仅对迭代次数固定或可预测的循环效果明显

4. 内联函数（Function Inlining）

定义：

将函数调用处直接替换为函数体，消除函数调用的开销（如参数传递、栈帧创建/销毁）。

示例：

// 优化前：函数调用有开销
int add(int a, int b) {
    return a + b;
}

int result = add(10, 20);

// 优化后：直接内联函数体
int result = 10 + 20;  // add函数体被直接替换到调用处

好处：

  • 消除函数调用的栈帧开销
  • 为后续优化（如常量折叠）创造条件

注意事项：

  • 仅对短小、调用频繁的函数效果明显
  • 过度内联会导致代码膨胀（程序体积变大）

总结

这些优化技术的核心目标是：减少运行时开销、提高执行效率。编译器会根据具体情况自动应用这些优化，无需开发者手动修改代码（但了解这些技术有助于编写更易优化的代码）。
