SpringCloudFeign转发请求头的小细节

约 277 字小于 1 分钟

JavaSpringCloud

2025-01-30

公司的某项目用的是新的技术栈，做的平台化，最外层就是采用SpringCloud作为网关，透传请求到业务中台。

这个错误与个人开发习惯有关，是个惯性错误。

然后呢，我在写一个接口的对接后，习惯性自测下，发现个问题，我请求SpringCloud项目，用了@FeignClient注解，这个注解的作用是调用对应HTTP接口。然后，就发现了请求头转发不过去。就很纳闷，这是为啥捏。

百度了一圈，都不是我想要的，仔细的琢磨了一下，发现了错误点。

我最近开始喜欢上了@Resource注解，因为，IDEA开发的时候，使用@Autowired注解，IDEA总是会飘黄，就很尴尬。所以，我注入Client的时候用的是@Resource注解。

换成@Autowired注解，就OK了，这俩的区别不用多说哈，@Autowired注解是Spring帮你管理这个bean，所以，你懂得。

嗯，拜拜~
