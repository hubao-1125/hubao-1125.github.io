GitHub推送代码时间过长问题

约 245 字小于 1 分钟

GitGitHub

2025-04-21

最近往GitHub推送代码的时候，有时候推送时间特别长，得好几分钟才能推上去。

但是有时候又直接报错Time out，这个错误是因为22端口不通。

如果22端口不通，则需要在.ssh目录下维护config文件。

这个文件不是所有人的.ssh目录下都有，没有的新建即可。

以下是config文件内容

Host github.com
    HostName ssh.github.com
    Port 443

新建完成之后，执行一下ssh -T git@github.com，看一下是否有返回

Hi 你的用户名! You've successfully authenticated, but GitHub does not provide shell access.

有这么个返回，就大功告成了，如果没有返回，则需要重新生成密钥，在GitHub网页端配置。

22端口不通的主要原因，可能是被防火墙给Ban了，防火墙给放开也行。
