什么是年轻代

约 414 字大约 1 分钟

JVM面试题

2025-12-29

年轻代（Young Generation）是Java堆内存的重要组成部分，主要用于存储新创建的对象，是JVM垃圾回收（GC）的高频区域。

核心特征与组成：

  1. 位置：位于JVM堆内存中，与老年代（Old Generation） 共同构成完整的堆内存。
  2. 区域划分：
    • Eden区（伊甸园）：新对象首先在此分配内存（约占年轻代80%空间）。
    • Survivor区：分为两个等大小的子区域（各占年轻代10%空间）：
      • From Survivor（S0）：当前存活对象的存放区
      • To Survivor（S1）：GC时用于复制存活对象的目标区
  3. 内存比例：典型比例为 Eden:S0:S1 = 8:1:1（可通过JVM参数调整）。

工作机制：

  1. 对象分配：新对象优先在Eden区分配内存。
  2. Minor GC（年轻代GC）：
    • 当Eden区满时触发，清理年轻代中的垃圾对象。
    • 采用复制算法：将Eden和From Survivor中的存活对象复制到To Survivor。
    • 每次GC后，存活对象的年龄计数+1。
  3. 对象晋升：
    • 当对象年龄达到阈值（默认15，可通过-XX:MaxTenuringThreshold调整），会从年轻代晋升到老年代。
    • 如果To Survivor空间不足，对象可能直接进入老年代。

设计目的：

  • 利用大部分对象生命周期短的特性（弱分代假说），通过频繁的Minor GC快速回收短命对象。
  • 复制算法在存活对象少的场景下效率极高，适合年轻代的回收需求。
