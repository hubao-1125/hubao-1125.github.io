不同版本的JDK的新特性

约 2410 字大约 8 分钟

JDKJava版本特性面试题

2026-01-05

引言

Java Development Kit (JDK) 是Java平台的核心组件，Oracle公司和开源社区不断地对其进行更新和改进。自JDK 8发布以来，Java生态系统经历了重大变化，采用了每6个月发布一个新版本的快速发布周期，同时每3年发布一个长期支持(LTS)版本。本文将详细介绍从JDK 8到最新版本的主要特性和改进。

JDK 8 (2014年3月)

JDK 8是Java历史上最重要的版本之一，引入了许多革命性的特性：

1. Lambda表达式

Lambda表达式允许将函数作为方法参数，使代码更加简洁和易读：

// 传统方式
Collections.sort(list, new Comparator<String>() {
    public int compare(String s1, String s2) {
        return s1.length() - s2.length();
    }
});

// Lambda表达式
Collections.sort(list, (s1, s2) -> s1.length() - s2.length());

2. Stream API

Stream API提供了一种高效的处理集合数据的方式，支持函数式编程风格：

List<String> filtered = list.stream()
                            .filter(s -> s.startsWith("a"))
                            .map(String::toUpperCase)
                            .sorted()
                            .collect(Collectors.toList());

3. Optional类

Optional类用于解决空指针异常问题，提供了一种更安全的处理可能为空值的方式：

Optional<String> optional = Optional.ofNullable(str);
optional.ifPresent(System.out::println);
String value = optional.orElse("default");

4. 新的日期时间API

引入了线程安全的新日期时间API，解决了旧API的许多问题：

LocalDate date = LocalDate.now();
LocalTime time = LocalTime.now();
LocalDateTime dateTime = LocalDateTime.now();
ZonedDateTime zonedDateTime = ZonedDateTime.now(ZoneId.of("Asia/Shanghai"));

5. 接口默认方法和静态方法

允许在接口中定义默认方法和静态方法，增强了接口的灵活性：

interface MyInterface {
    default void defaultMethod() {
        System.out.println("Default method");
    }
    
    static void staticMethod() {
        System.out.println("Static method");
    }
}

6. Nashorn JavaScript引擎

替代了Rhino引擎，提供了更快的JavaScript执行速度。

JDK 9 (2017年9月)

JDK 9引入了模块化系统和其他重要特性：

1. Java Platform Module System (JPMS)

也称为Project Jigsaw，引入了模块化的概念，使代码更易于维护和扩展：

module com.example.module {
    requires java.base;
    exports com.example.module.api;
}

2. JShell

引入了交互式编程环境，允许开发者快速测试Java代码片段：

jshell>
2 + 3
$1 ==> 5

jshell>
String s = "Hello";

jshell>
System.out.println(s);
Hello

3. 接口私有方法

允许在接口中定义私有方法，提高代码复用性：

interface MyInterface {
    default void publicMethod() {
        privateMethod();
    }
    
    private void privateMethod() {
        System.out.println("Private method");
    }
}

4. 增强的Stream API

添加了新的方法，如takeWhile、dropWhile、ofNullable等：

Stream.of(1, 2, 3, 4, 5)
      .takeWhile(n -> n < 4)
      .forEach(System.out::println); // 输出1, 2, 3

5. 增强的Optional类

添加了ifPresentOrElse、or、stream等方法：

optional.ifPresentOrElse(
    System.out::println,
    () -> System.out.println("Value not present")
);

JDK 10 (2018年3月)

JDK 10主要引入了局部变量类型推断：

1. 局部变量类型推断 (var)

允许编译器根据上下文推断局部变量的类型：

var list = new ArrayList<String>(); // 推断为ArrayList<String>
var map = new HashMap<String, Integer>(); // 推断为HashMap<String, Integer>
var stream = list.stream(); // 推断为Stream<String>

2. 改进的GC

引入了G1 GC的并行全垃圾回收，提高了垃圾回收效率。

3. 其他改进

  • 线程局部变量的存储优化
  • 统一的JVM日志系统
  • 根证书颁发机构(CA)证书存储

JDK 11 (2018年9月) - LTS版本

JDK 11是一个长期支持版本，引入了许多重要特性：

1. 字符串API增强

"   hello   ".strip(); // 去除首尾空白，类似于trim()但支持Unicode
"hello".repeat(3); // 重复字符串："hellohellohello"
"hello".isBlank(); // 检查字符串是否为空或只包含空白字符
"hello\nworld".lines().count(); // 计算行数：2

2. HttpClient API

引入了新的HttpClient API，支持异步和同步请求：

HttpClient client = HttpClient.newHttpClient();
HttpRequest request = HttpRequest.newBuilder()
        .uri(URI.create("https://example.com"))
        .build();

// 同步请求
HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());

// 异步请求
client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
      .thenApply(HttpResponse::body)
      .thenAccept(System.out::println);

3. ZGC垃圾收集器 (实验性)

引入了低延迟垃圾收集器ZGC，目标是将停顿时间控制在10毫秒以内：

java -XX:+UnlockExperimentalVMOptions -XX:+UseZGC MainClass

4. Epsilon垃圾收集器 (实验性)

一种不执行任何垃圾回收的垃圾收集器，用于性能测试和特定场景。

5. 移除Java EE和CORBA模块

移除了java.xml.ws、java.xml.bind、java.activation、java.xml.ws.annotation、java.corba等模块。

JDK 12 (2019年3月)

1. Switch表达式 (预览)

增强了switch语句，使其可以作为表达式使用：

int result = switch (day) {
    case MONDAY, FRIDAY -> 6;
    case TUESDAY -> 7;
    default -> 8;
};

2. Shenandoah GC (实验性)

引入了Shenandoah垃圾收集器，目标是实现低停顿时间的并发垃圾回收。

3. 微基准测试套件

引入了JMH微基准测试套件，用于更准确地测量Java代码性能。

JDK 13 (2019年9月)

1. Switch表达式增强 (预览)

添加了yield语句，用于从switch表达式中返回值：

int result = switch (day) {
    case MONDAY, FRIDAY -> 6;
    case TUESDAY -> 7;
    default -> {
        yield 8;
    }
};

2. 文本块预览

引入了文本块功能，允许更方便地处理多行字符串：

String html = """
    <html>
        <body>
            <p>Hello, world</p>
        </body>
    </html>
""";

JDK 14 (2020年3月)

1. Records (预览)

引入了Records，用于创建不可变数据类：

record Person(String name, int age) {
    // 自动生成构造函数、equals()、hashCode()、toString()等方法
}

Person person = new Person("John", 30);
System.out.println(person.name()); // John
System.out.println(person.age()); // 30

2. 模式匹配instanceof (预览)

增强了instanceof操作符，允许直接将匹配的对象转换为相应类型：

if (obj instanceof String s) {
    // 直接使用s，无需额外转换
    System.out.println(s.length());
}

3. 文本块标准化

文本块功能从预览状态变为标准功能。

JDK 15 (2020年9月)

1. Sealed Classes (密封类) 预览

允许限制哪些类可以继承或实现特定类或接口：

sealed class Shape permits Circle, Rectangle, Triangle {
    // 类定义
}

final class Circle extends Shape {}
final class Rectangle extends Shape {}
final class Triangle extends Shape {}

2. Records 正式化

Records功能从预览状态变为标准功能。

3. Hidden Classes

引入了隐藏类功能，用于动态生成的类，提高性能和安全性。

JDK 16 (2021年3月)

1. Records 正式版

Records功能成为正式功能。

2. 模式匹配instanceof 正式版

模式匹配instanceof功能成为正式功能。

3. Sealed Classes 预览

密封类功能进入第二个预览阶段。

4. Vector API (孵化)

引入了Vector API，用于进行向量计算，提高性能。

5. Foreign Linker API (孵化)

用于与本地代码进行交互，替代JNI。

JDK 17 (2021年9月) - LTS版本

JDK 17是一个长期支持版本：

1. Sealed Classes 正式版

密封类功能成为正式功能。

2. 移除实验性的AOT和JIT编译器

移除了实验性的Ahead-of-Time (AOT) 和Just-In-Time (JIT) 编译器。

3. 增强的伪随机数生成器

添加了新的伪随机数生成器接口和实现。

4. Switch表达式增强

switch表达式成为标准功能。

JDK 18 (2022年3月)

1. UTF-8作为默认字符集

将UTF-8设置为Java API的默认字符集。

2. Simple Web Server

引入了简单的Web服务器，用于测试和开发：

java -m jdk.httpserver --port 8000

3. Vector API 第二次孵化

Vector API进入第二个孵化阶段。

4. Pattern Matching for Switch (预览)

增强了switch语句，支持模式匹配：

static String formatterPatternSwitch(Object o) {
    return switch (o) {
        case Integer i -> String.format("int %d", i);
        case Long l -> String.format("long %d", l);
        case Double d -> String.format("double %f", d);
        case String s -> String.format("String %s", s);
        default -> o.toString();
    };
}

JDK 19 (2022年9月)

1. Virtual Threads (预览)

引入了虚拟线程，轻量级线程，支持高并发：

Runnable task = () -> System.out.println("Hello, Virtual Thread");
Thread virtualThread = Thread.ofVirtual().start(task);

2. Structured Concurrency (预览)

引入了结构化并发API，简化并发编程：

Response handle() throws ExecutionException, InterruptedException {
    try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
        Supplier<String> user = scope.fork(() -> findUser());
        Supplier<Integer> order = scope.fork(() -> fetchOrder());

        scope.join(); // 等待所有任务完成
        scope.throwIfFailed(); // 抛出第一个异常

        // 两个结果都成功
        return new Response(user.get(), order.get());
    }
}

3. Record Patterns (预览)

增强了模式匹配，支持对Record进行解构：

if (obj instanceof Person(String name, int age)) {
    System.out.println("Name: " + name + ", Age: " + age);
}

JDK 20 (2023年3月)

1. Virtual Threads (预览)

虚拟线程功能进入第二个预览阶段。

2. Structured Concurrency (预览)

结构化并发API进入第二个预览阶段。

3. Pattern Matching for Switch (预览)

switch模式匹配功能进入第三个预览阶段。

JDK 21 (2023年9月) - LTS版本

1. Virtual Threads (正式版)

虚拟线程功能成为正式功能。

2. Structured Concurrency (预览)

结构化并发API进入第三个预览阶段。

3. Record Patterns (预览)

Record Patterns功能进入第二个预览阶段。

4. Pattern Matching for Switch (正式版)

switch模式匹配功能成为正式功能。

5. Sequenced Collections

引入了新的集合接口，用于处理有序集合：

SequencedCollection<String> list = new ArrayList<>();
list.addFirst("a");
list.addLast("b");
SequencedSet<String> set = new LinkedHashSet<>();
SequencedMap<String, Integer> map = new LinkedHashMap<>();

总结

自JDK 8以来，Java平台经历了快速发展，每个版本都引入了许多重要特性和改进。从函数式编程支持(Lambda表达式、Stream API)到模块化系统、新的日期时间API、虚拟线程等，Java不断适应现代软件开发的需求。

对于企业开发者来说，选择合适的JDK版本非常重要。通常建议使用LTS版本(JDK 8, JDK 11, JDK 17, JDK 21)以获得长期支持和稳定性。而对于希望尝试最新特性的开发者，可以使用非LTS版本来体验Java的最新进展。

随着Java的不断发展，我们可以期待更多令人兴奋的特性和改进，使Java在未来继续保持其在企业级开发中的领先地位。
